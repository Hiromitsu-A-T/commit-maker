# Code of Conduct / 行動規範

Commit Maker is maintained as a small open-source project. This code of conduct
keeps collaboration respectful, practical, and safe for users and contributors.

Commit Maker は小規模なオープンソースプロジェクトとして運営されています。
この行動規範は、利用者と貢献者が安全かつ建設的に協力できる状態を保つためのものです。

## Expected Behavior / 期待される行動

- 敬意を持って、建設的にやり取りしてください。
- 技術的な問題とプロジェクトの目的に集中してください。
- 再現手順、背景、判断材料をできるだけ具体的に共有してください。
- メンテナーと貢献者の時間を尊重してください。
- 日本語・英語のどちらでも構いません。読み手に伝わるよう、必要に応じて補足してください。

- Be respectful and constructive.
- Keep discussions focused on the technical problem and the project.
- Share reproduction steps, context, and decision points clearly.
- Respect maintainers' and contributors' time.
- Japanese and English are both welcome. Add context when it helps others
  understand the issue.

## Unacceptable Behavior / 許容されない行動

- ハラスメント、脅迫、個人攻撃、継続的な妨害。
- 差別的な言動。
- 本人の許可なく個人情報を公開すること。
- API キー、トークン、非公開差分、機密ログを公開すること。
- スパム、虚偽の情報、プロジェクトと無関係な宣伝。

- Harassment, threats, personal attacks, or sustained disruption.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Sharing API keys, tokens, private diffs, or sensitive logs.
- Spam, deceptive behavior, or unrelated promotion.

## Reporting / 報告

通常の不具合や要望は GitHub Issues を使ってください。
行動規範に関する非公開の相談は、メンテナーの GitHub profile に記載された連絡先から
連絡してください。

For normal project issues, use GitHub Issues. For private conduct concerns,
contact the maintainer through the contact method listed on the maintainer's
GitHub profile.

報告には状況を理解できるだけの情報を含めてください。ただし、秘密情報や非公開データは、
必要な場合を除き含めないでください。

Include enough context to understand what happened, but do not include secrets
or private data unless they are necessary to evaluate the report.

## Enforcement / 対応

メンテナーは、コメントの削除、Issue / Pull Request のクローズ、ユーザーのブロックなど、
プロジェクトと参加者を守るために必要な対応を行うことがあります。

Maintainers may remove comments, close issues or pull requests, block users, or
take other reasonable action to protect the project and its contributors.
