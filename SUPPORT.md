# Support / サポート

Use the following channels for Commit Maker support.
Commit Maker のサポートは、以下の方法で受け付けます。

日本語・英語のどちらでも構いません。
Japanese and English are both welcome.

## Questions and Usage Help / 質問・使い方

GitHub Discussions が利用できる場合は Discussions を、利用できない場合は GitHub
Issues を使ってください。

Use GitHub Discussions if available. If discussions are not available, open a
GitHub Issue.

Include / 含めてほしい情報:

- Commit Maker version
- VS Code version
- Operating system
- Provider in use: Gemini, OpenAI, Claude, or Local
- What you expected and what happened

API キー、Personal Access Token、非公開リポジトリの差分、その他の秘密情報は
含めないでください。公開するログは必ず redaction してください。

Do not include API keys, personal access tokens, private repository diffs, or
other secrets. Redact logs before sharing them publicly.

## Bug Reports / バグ報告

バグ報告には bug report template を使ってください。最小の再現手順が最も役立ちます。

Use the bug report issue template. A minimal reproduction is the most useful
form of report.

Local provider の問題では、選択したモデル名、runtime platform、model/runtime download
が完了したかどうかを含めてください。

For Local provider issues, include the selected model name, runtime platform,
and whether the model/runtime download completed successfully.

## Feature Requests / 機能要望

feature request template を使ってください。先に workflow 上の問題を書き、その後に
期待する挙動を書いてください。

Use the feature request issue template. Explain the workflow problem first, then
the proposed behavior.

## Security Issues / セキュリティ

脆弱性は public Issue では報告しないでください。[`SECURITY.md`](SECURITY.md) に
従ってください。

Do not report vulnerabilities in public issues. Follow
[`SECURITY.md`](SECURITY.md) instead.
